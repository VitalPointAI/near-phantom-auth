Land BACKUP-02 end-to-end:

1. Extend `DatabaseAdapter` with optional `updatePasskeyBackedUp(credentialId, backedUp): Promise<void>` (per project constraint: new adapter methods MUST be optional with internal fallback).
2. Implement that method in the Postgres adapter at `src/server/db/adapters/postgres.ts`, mirroring `updatePasskeyCounter`.

5. Add a BACKUP-02 BS-bit-flip-on-login supertest assertion that proves the response carries the FRESH value (not the stored one).

Purpose: BS bit can flip 0→1 over a credential's lifetime. The library must observe and surface the freshly-read value on every login, and persist it so subsequent flows have an accurate cache. The persistence is best-effort (optional adapter method); the response surfacing is mandatory.
