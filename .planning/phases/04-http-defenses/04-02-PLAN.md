  <action>
1. In `src/server/router.ts`:
   - Add `import { rateLimit } from 'express-rate-limit';` at the top
   - Inside `createRouter()`, after extracting config fields, create two limiter instances:

   ```typescript
   const authRateConfig = config.rateLimiting?.auth ?? {};
   const recoveryRateConfig = config.rateLimiting?.recovery ?? {};

   const authLimiter = rateLimit({
     windowMs: authRateConfig.windowMs ?? 15 * 60 * 1000,
     limit: authRateConfig.limit ?? 20,
     standardHeaders: 'draft-8',
     legacyHeaders: false,
     handler: (_req, res, _next, options) => {
       log.warn({ limit: options.limit }, 'auth rate limit exceeded');
       res.status(429).json({ error: 'Too many requests. Please try again later.' });
     },
   });

   const recoveryLimiter = rateLimit({
     windowMs: recoveryRateConfig.windowMs ?? 60 * 60 * 1000,
     limit: recoveryRateConfig.limit ?? 5,
     standardHeaders: 'draft-8',
     legacyHeaders: false,
     handler: (_req, res, _next, options) => {
       log.warn({ limit: options.limit }, 'recovery rate limit exceeded');
       res.status(429).json({ error: 'Too many recovery attempts. Please try again later.' });
     },
   });
   ```

   - Apply `authLimiter` as the FIRST middleware argument (before `json()` or `async (req, res)`) on these routes:
     - `router.post('/register/start', authLimiter, ...)`
     - `router.post('/register/finish', authLimiter, ...)`
     - `router.post('/login/start', authLimiter, ...)`
     - `router.post('/login/finish', authLimiter, ...)`
     - `router.post('/logout', authLimiter, ...)`

   - Apply `recoveryLimiter` as the FIRST middleware argument on these routes:
     - `router.post('/recovery/wallet/link', recoveryLimiter, ...)`
     - `router.post('/recovery/wallet/verify', recoveryLimiter, ...)`
     - `router.post('/recovery/wallet/start', recoveryLimiter, ...)`
     - `router.post('/recovery/wallet/finish', recoveryLimiter, ...)`
     - `router.post('/recovery/ipfs/setup', recoveryLimiter, ...)`
     - `router.post('/recovery/ipfs/recover', recoveryLimiter, ...)`

   - Do NOT apply any limiter to `router.get('/session', ...)` — it is read-only.

2. In `src/server/oauth/router.ts`:
   - Add `import { rateLimit } from 'express-rate-limit';`
   - Inside `createOAuthRouter()`, create an authLimiter using the same pattern (reading from `config.rateLimiting?.auth`):

   ```typescript
   const authRateConfig = config.rateLimiting?.auth ?? {};

       log.warn({ limit: options.limit }, 'auth rate limit exceeded');
       res.status(429).json({ error: 'Too many requests. Please try again later.' });
     },
   });
   ```

   - Apply `authLimiter` to:
     - `router.get('/:provider/start', authLimiter, ...)`
     - `router.post('/:provider/callback', authLimiter, ...)`
     - `router.post('/:provider/link', authLimiter, ...)`

   NOTE: Each router creates its own limiter instances. This means the auth limiter in router.ts and oauth/router.ts have SEPARATE counters. This is intentional — a consumer mounting them on different paths gets independent rate tracking.
