```typescript
import { doubleCsrf } from 'csrf-csrf';
import cookieParser from 'cookie-parser';

const { generateToken, doubleCsrfProtection } = doubleCsrf({
  getSecret: () => config.csrf!.secret,
  cookieName: '__Host-csrf',
  cookieOptions: {
    httpOnly: true,
    sameSite: 'strict',
    secure: true,
    path: '/',
  },
  ignoredMethods: ['GET', 'HEAD', 'OPTIONS'],
  getTokenFromRequest: (req) => req.headers['x-csrf-token'] as string,
  skipCsrfProtection: (req) => {
    return /^\/oauth\/[^/]+\/callback$/.test(req.path);
  },
});

// Order matters: cookieParser() BEFORE doubleCsrfProtection
router.use(cookieParser());
router.use(doubleCsrfProtection);

// Token endpoint (GET — not CSRF-protected due to ignoredMethods)
router.get('/csrf-token', (req, res) => {
  res.json({ token: generateToken(req, res) });
});
```

   - Inside `createRouter()`, AFTER creating the rate limiters and BEFORE defining routes, add conditional CSRF setup:
     ```typescript
     // CSRF protection (opt-in via config.csrf)
     if (config.csrf) {
       const { generateToken, doubleCsrfProtection } = doubleCsrf({
         getSecret: () => config.csrf!.secret,
         cookieName: '__Host-csrf',
         cookieOptions: {
           httpOnly: true,
           sameSite: 'strict',
           secure: true,

         res.json({ token: generateToken(req, res) });
       });

       log.info('CSRF protection enabled');
     }
     ```
   - NOTE: The cookieParser() and doubleCsrfProtection are applied via `router.use()` BEFORE any route definitions. This means all POST routes on this router are CSRF-protected when enabled. GET routes are exempt via `ignoredMethods`.

2. In `src/server/oauth/router.ts`:
   - Add imports:
     ```typescript
     import { doubleCsrf } from 'csrf-csrf';
     import cookieParser from 'cookie-parser';
     ```
   - Inside `createOAuthRouter()`, AFTER creating the authLimiter and BEFORE defining routes, add conditional CSRF with OAuth exemption:
     ```typescript
     // CSRF protection (opt-in via config.csrf) with OAuth callback exemption
     if (config.csrf) {
       const { doubleCsrfProtection } = doubleCsrf({
         getSecret: () => config.csrf!.secret,
         cookieName: '__Host-csrf',
         cookieOptions: {
           httpOnly: true,
           sameSite: 'strict',
           secure: true,
           path: '/',
         },
         ignoredMethods: ['GET', 'HEAD', 'OPTIONS'],
         getTokenFromRequest: (req) => req.headers['x-csrf-token'] as string,
         // OAuth callback arrives cross-origin from provider; cannot carry CSRF cookie.
         // OAuth flow uses state parameter validation as its own CSRF defense.
         skipCsrfProtection: (req) => {
           return /^\/[^/]+\/callback$/.test(req.path);
         },
       });

       router.use(cookieParser());
       router.use(doubleCsrfProtection);

       log.info('CSRF protection enabled for OAuth router (callback exempt)');
     }
     ```
   - NOTE: The OAuth callback regex is `^\/[^/]+\/callback$` (not `^\/oauth\/...`) because the OAuth router is mounted as a sub-router — `req.path` is relative to the mount point, so it sees `/:provider/callback` not `/oauth/:provider/callback`.

3. In `src/server/oauth/router.ts`, add the INFRA-05 cookie-parser guard inside the `/:provider/callback` POST handler, as the FIRST check inside the try block (before any `req.cookies` access):
   ```typescript
   // INFRA-05: Detect missing cookie-parser middleware
   if (req.cookies === undefined) {
     log.error(
       'OAuth callback received request without req.cookies. ' +
       'Mount cookie-parser middleware before the OAuth router: app.use(cookieParser())'
     );
     return res.status(500).json({
       error: 'Server configuration error: cookie-parser middleware is required',
     });
   }
   ```
   This guard fires even when CSRF is disabled (since CSRF auto-adds cookieParser, but a consumer might disable CSRF and also forget cookie-parser).
