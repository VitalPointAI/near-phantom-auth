<!-- From Plan 01, each manager has a `log` child logger available. The replacement pattern: -->

Level mapping:
- console.log   -> log.info
- console.warn  -> log.warn
- console.error -> log.error

Error serialization pattern (pino auto-serializes err with stack trace):
  log.error({ err: error }, 'Message');
  NOT: log.error('Message:', error);

Structured field pattern (first arg is object, second is message string):
  log.info({ nearAccountId, network: this.networkId }, 'Creating NEAR account');
  NOT: log.info('Creating NEAR account:', { nearAccountId });

Fields that MUST NEVER appear as log properties:
- treasuryPrivateKey
- derivationPath
- derivationSalt
- mpcPublicKey
- sessionSecret
- raw request body (req.body)

Fields that are SAFE to log:
- nearAccountId, network/networkId, mpcContractId
- userId, codenameStyle
- cid (IPFS content ID)
- provider (OAuth provider name)
- error objects (via { err: error })
- HTTP status codes, route paths
</interfaces>
