<objective>
Update documentation so consumers understand how to use the new session metadata policy and what each mode means for anonymity. This closes the gap identified in the security review: NEAR account IDs are not linked to user PII, but raw session IP/user-agent metadata can still be operationally linkable unless omitted or transformed.
</objective>
