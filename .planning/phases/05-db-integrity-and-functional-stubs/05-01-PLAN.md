1. In `src/types/index.ts`, add three optional methods to the `DatabaseAdapter` interface (after `updateSessionExpiry?` on line 197):

```typescript
  /** Optional: wrap multiple operations in a database transaction.
   *  If not implemented, operations execute sequentially (no atomicity guarantee). */
  transaction?<T>(fn: (tx: DatabaseAdapter) => Promise<T>): Promise<T>;

  /** Optional: delete a user by ID. Passkeys cascade via FK; sessions and recovery must be deleted first. */
  deleteUser?(userId: string): Promise<void>;

  /** Optional: delete all recovery data for a user. */
  deleteRecoveryData?(userId: string): Promise<void>;
```

2. In `src/server/db/adapters/postgres.ts`, implement all three methods in the returned adapter object:

**transaction():** Follow the existing `createOAuthUser` pattern (lines 237-267). Use `pool.connect()` to get a client, `BEGIN`/`COMMIT`/`ROLLBACK`. Build a `txAdapter` object that proxies the subset of DatabaseAdapter methods needed by the registration flow (createUser, createPasskey, createSession, storeChallenge, deleteChallenge) but routes queries through the transaction client instead of the pool. The txAdapter should implement all DatabaseAdapter methods that the `fn` callback might call — at minimum: createUser, createPasskey, createSession. For methods not needed in a transaction context, forward to the main adapter.

Implementation detail for txAdapter: Create a helper function `buildClientAdapter(client: PoolClient): DatabaseAdapter` that duplicates the SQL from the main adapter's createUser/createPasskey/createSession but uses `client.query()` instead of `(await getPool()).query()`. The helper only needs to implement methods used in the registration transaction — other methods can throw `new Error('Not available in transaction context')`.

**deleteUser():** `DELETE FROM anon_users WHERE id = $1` — passkeys cascade via FK; caller must delete sessions and recovery first.

**deleteRecoveryData():** `DELETE FROM anon_recovery WHERE user_id = $1`.

IMPORTANT: The project decision is "make new DatabaseAdapter methods optional with internal fallbacks" — all three methods use `?` in the interface.
