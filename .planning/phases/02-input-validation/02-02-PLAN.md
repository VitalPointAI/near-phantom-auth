<task type="auto">
  <name>Task 1: Wire validation into router.ts (10 POST handlers)</name>
  <files>src/server/router.ts</files>
  <action>
    1. Add imports at top of router.ts:
       ```typescript
       import { validateBody } from './validation/validateBody.js';
       import {
         registerStartBodySchema,
         registerFinishBodySchema,
         loginStartBodySchema,
         loginFinishBodySchema,
         logoutBodySchema,
         walletLinkBodySchema,
         walletVerifyBodySchema,
         walletStartBodySchema,
         walletFinishBodySchema,
         ipfsSetupBodySchema,
         ipfsRecoverBodySchema,
       } from './validation/schemas.js';
       ```

    2. For each POST handler, add `validateBody` call as the FIRST operation inside the try block, BEFORE any business logic. Use the parsed `body` result instead of `req.body` destructuring. Pattern:
       ```typescript
       router.post('/register/finish', async (req: Request, res: Response) => {
         try {
           const body = validateBody(registerFinishBodySchema, req, res);
           if (!body) return;
           const { challengeId, response, tempUserId, codename } = body;
           // ... rest of handler unchanged
       ```

    3. Route-by-route wiring:
       - POST /register/start: `validateBody(registerStartBodySchema, req, res)` — schema is empty object, no fields to destructure from body (codename generated server-side)
       - POST /register/finish: Replace `const { challengeId, response, tempUserId, codename } = req.body` AND the `if (!challengeId || !response || !tempUserId || !codename)` guard with `validateBody(registerFinishBodySchema, req, res)`
       - POST /login/start: Replace `const { codename } = req.body` with `validateBody(loginStartBodySchema, req, res)`, destructure codename from body result
       - POST /login/finish: Replace `const { challengeId, response } = req.body` AND the `if (!challengeId || !response)` guard
       - POST /logout: `validateBody(logoutBodySchema, req, res)` — no fields used from body
       - POST /recovery/wallet/link: `validateBody(walletLinkBodySchema, req, res)` — no body fields used
       - POST /recovery/wallet/verify: Replace `const { signature, challenge, walletAccountId } = req.body` AND the `if (!signature || !challenge || !walletAccountId)` guard
       - POST /recovery/wallet/start: `validateBody(walletStartBodySchema, req, res)` — no body fields used
       - POST /recovery/wallet/finish: Replace `const { signature, challenge, nearAccountId } = req.body` AND the `if (!signature || !challenge || !nearAccountId)` guard
       - POST /recovery/ipfs/setup: Replace `const { password } = req.body` AND the `if (!password)` guard
       - POST /recovery/ipfs/recover: Replace `const { cid, password } = req.body` AND the `if (!cid || !password)` guard

    4. DELETE all manual `if (!field)` guards that are now redundant — the schema enforces required fields with `z.string().min(1)`.

    5. Keep all other business logic (codename validation via isValidCodename, password strength validation via ipfsRecovery.validatePassword, etc.) exactly as-is. Schema validation handles structural correctness; business validation remains in handlers.
