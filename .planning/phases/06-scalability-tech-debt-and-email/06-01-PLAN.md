   - deleteOAuthState: DELETE FROM oauth_state WHERE state = $1

4. Implement cleanExpiredChallenges: DELETE FROM anon_challenges WHERE expires_at < NOW(). Return rowCount.

5. Implement cleanExpiredOAuthStates: DELETE FROM oauth_state WHERE expires_at < NOW(). Return rowCount.

6. Rewrite getOAuthUserById to use a single JOIN query:
   ```sql
   SELECT u.*, p.provider, p.provider_id, p.email AS p_email,
          p.name AS p_name, p.avatar_url AS p_avatar_url, p.connected_at
   FROM oauth_users u
   LEFT JOIN oauth_providers p ON p.user_id = u.id
   WHERE u.id = $1
   ```
   Aggregate multiple rows into a single OAuthUser with providers[] array. If first row is null for user fields, return null. Filter out rows where p.provider is null (user with zero providers).

7. Rewrite getOAuthUserByEmail similarly with LEFT JOIN, WHERE u.email = $1.

8. Rewrite getOAuthUserByProvider with INNER JOIN (provider row must exist):
   ```sql
   SELECT u.*, p.provider, p.provider_id, p.email AS p_email,
          p.name AS p_name, p.avatar_url AS p_avatar_url, p.connected_at,
          p2.provider AS p2_provider, p2.provider_id AS p2_provider_id,
          p2.email AS p2_email, p2.name AS p2_name, p2.avatar_url AS p2_avatar_url,
          p2.connected_at AS p2_connected_at
   FROM oauth_providers p
   JOIN oauth_users u ON u.id = p.user_id
   LEFT JOIN oauth_providers p2 ON p2.user_id = u.id
   WHERE p.provider = $1 AND p.provider_id = $2
   ```
   Actually, simpler approach: use a subquery or two-step single query. The cleanest pattern per the research is:
   ```sql
   SELECT u.*, p.provider, p.provider_id, p.email AS p_email,
          p.name AS p_name, p.avatar_url AS p_avatar_url, p.connected_at
   FROM oauth_users u
   JOIN oauth_providers p ON p.user_id = u.id
   WHERE u.id = (
     SELECT user_id FROM oauth_providers WHERE provider = $1 AND provider_id = $2
   )
   ```
   This returns all providers for the user in one query (the subquery finds the user_id, then the main query gets user + all providers).

9. Extract a shared helper function `mapOAuthUserRows(rows)` that takes the JOIN result rows and returns OAuthUser | null, to avoid duplicating the row-to-object mapping in all three methods.

   ```typescript
   if (db.getOAuthState) {
     const oauthState = await db.getOAuthState(state);

     if (oauthState.expiresAt < new Date()) {
       await db.deleteOAuthState?.(state);
       return null;
     }
     await db.deleteOAuthState?.(state); // Atomic consume — delete on read

     const oauthState = stateStore.get(state);
     if (!oauthState) return null;
     if (oauthState.expiresAt < new Date()) {
       stateStore.delete(state);
       return null;
     }
     stateStore.delete(state);
     return oauthState;
   }
   ```
