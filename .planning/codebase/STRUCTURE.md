## Directory Purposes

**`src/types/`:**
- Purpose: Central type definitions shared across server and client
- Contains: All interfaces (`AnonAuthConfig`, `DatabaseAdapter`, `AnonUser`, `OAuthUser`, `Session`, `Passkey`, `Challenge`, etc.), type aliases, WebAuthn JSON types, Express augmentation
- Key files: `src/types/index.ts` (single file, ~428 lines)

**`src/server/`:**
- Purpose: Server-side SDK for Express applications
- Contains: Factory function, domain managers, routers, middleware, database adapters
- Key files: `src/server/index.ts` (main entry), `src/server/router.ts` (API endpoints), `src/server/mpc.ts` (NEAR account creation)

**`src/server/db/adapters/`:**
- Purpose: Database adapter implementations
- Contains: PostgreSQL adapter with full SQL schema
- Key files: `src/server/db/adapters/postgres.ts` (609 lines, includes `POSTGRES_SCHEMA` DDL)

**`src/server/oauth/`:**
- Purpose: OAuth provider integration (Google, GitHub, Twitter/X)
- Contains: OAuth manager with PKCE support, dedicated Express router
- Key files: `src/server/oauth/index.ts` (manager), `src/server/oauth/router.ts` (routes)

**`src/server/recovery/`:**
- Purpose: Account recovery mechanisms
- Contains: Wallet-based recovery (NEAR on-chain), IPFS+password recovery (encrypted backup)
- Key files: `src/server/recovery/wallet.ts`, `src/server/recovery/ipfs.ts`

**`src/client/`:**
- Purpose: Browser-side SDK for React and vanilla JS applications
- Contains: React hooks/providers, HTTP API client, browser WebAuthn utilities
- Key files: `src/client/hooks/useAnonAuth.tsx` (main React hook), `src/client/api.ts` (API client)

**`src/client/hooks/`:**
- Purpose: React Context providers and hooks
- Contains: `useAnonAuth` for passkey auth, `useOAuth` for OAuth auth
- Key files: `src/client/hooks/useAnonAuth.tsx`, `src/client/hooks/useOAuth.tsx`

**`src/webauthn/`:**
- Purpose: Lightweight WebAuthn-only export for frameworks managing their own sessions (Next.js)
- Contains: Re-exports from `src/server/webauthn.ts`
- Key files: `src/webauthn/index.ts`

**`dist/`:**
- Purpose: Compiled output (ESM + CJS + type declarations)
- Generated: Yes (by `tsup`)
- Committed: Yes (included in `files` for npm publish)

## Key File Locations

**Entry Points:**
- `src/index.ts`: Package root - type re-exports only
- `src/server/index.ts`: Server SDK entry - `createAnonAuth()` factory + all server exports
- `src/client/index.ts`: Client SDK entry - React hooks + API client + passkey utils
- `src/webauthn/index.ts`: Standalone WebAuthn entry

**Configuration:**
- `package.json`: Package manifest with 4 export maps (`.`, `./server`, `./client`, `./webauthn`)
- `tsconfig.json`: TypeScript config (ES2022, JSX react-jsx, strict=false)
- `tsup.config.ts`: Build config (4 entries, ESM+CJS, dts, sourcemaps)

**Core Logic:**
- `src/server/mpc.ts`: NEAR MPC account creation, implicit accounts, treasury funding (566 lines)
- `src/server/passkey.ts`: WebAuthn passkey registration/authentication with DB (307 lines)
- `src/server/router.ts`: All passkey + recovery REST endpoints (547 lines)
- `src/server/oauth/router.ts`: All OAuth REST endpoints (340 lines)
- `src/server/session.ts`: Session lifecycle with HMAC-signed cookies (209 lines)
- `src/server/recovery/ipfs.ts`: AES-256-GCM encrypted IPFS backup (398 lines)
- `src/server/db/adapters/postgres.ts`: Full PostgreSQL adapter + schema DDL (609 lines)

**Testing:**
- No test files exist in the codebase currently

## Naming Conventions

**Files:**
- `camelCase.ts` for modules: `passkey.ts`, `codename.ts`, `session.ts`
- `camelCase.tsx` for React components: `useAnonAuth.tsx`, `useOAuth.tsx`
- `index.ts` for barrel exports in each directory

**Directories:**
- `lowercase` single-word: `server/`, `client/`, `types/`, `recovery/`, `hooks/`, `webauthn/`
- `db/adapters/` for database adapter implementations

**Exports:**
- Factory functions: `create` prefix - `createAnonAuth()`, `createPasskeyManager()`, `createSessionManager()`, `createOAuthManager()`, `createRouter()`, etc.
- React hooks: `use` prefix - `useAnonAuth()`, `useOAuth()`, `useOAuthCallback()`
- React providers: `Provider` suffix - `AnonAuthProvider`, `OAuthProvider`
- Interfaces: PascalCase with descriptive suffixes - `SessionManager`, `PasskeyManager`, `MPCAccountManager`, `DatabaseAdapter`
- Config interfaces: `Config` suffix - `AnonAuthConfig`, `PasskeyConfig`, `MPCConfig`, `SessionConfig`

## Where to Add New Code

**New Authentication Method (e.g., email magic link):**
- Manager: `src/server/email.ts` (create `EmailManager` interface + factory)
- Router: `src/server/email/router.ts` or add routes to `src/server/router.ts`
- Client hook: `src/client/hooks/useEmail.tsx`
- Types: Add to `src/types/index.ts`
- Wire up in `src/server/index.ts` (`createAnonAuth()` factory)
- Export from `src/client/index.ts`

**New Database Adapter (e.g., SQLite, MongoDB):**
- Implementation: `src/server/db/adapters/sqlite.ts`
- Must implement `DatabaseAdapter` interface from `src/types/index.ts`
- Export from `src/server/index.ts`
- Add to database type union in `DatabaseConfig.type`

**New OAuth Provider (e.g., Discord, Apple):**
- Add provider config to `OAuthProviderConfig` in `src/server/oauth/index.ts`
- Add auth URL generation, token exchange, and profile fetching in `createOAuthManager()`
- Add to `OAuthConfig` interface in `src/types/index.ts`
- Update `OAuthProvider['provider']` union type in `src/types/index.ts`

**New Recovery Method:**
- Implementation: `src/server/recovery/newmethod.ts`
- Create manager interface + factory function
- Add routes in `src/server/router.ts`
- Add client API methods in `src/client/api.ts`
- Add to `RecoveryConfig` in `src/types/index.ts`

**New Client Utility:**
- Framework-agnostic: `src/client/utilname.ts`
- React hook: `src/client/hooks/useUtilName.tsx`
- Export from `src/client/index.ts`

**New Standalone Export Entry:**
- Create: `src/newentry/index.ts`
- Add to `tsup.config.ts` entry points
- Add to `package.json` exports map
